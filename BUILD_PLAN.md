# Build Plan

## 0. 准备环境

当前目录已经是一个 Git 仓库，并且已经有了第一版静态网站文件。

需要确认或安装：

- Node.js
- npm 或 pnpm
- GitHub 账号
- Vercel 账号
- Supabase 账号
- 一个以后要绑定的域名

当前机器能找到 `node`，但暂时找不到 `npm`。下一步需要先解决包管理器问题，才能初始化 Next.js 项目。

## 1. 初始化网站

目标：本地可以打开一个网站页面。

计划：

1. 创建 Next.js 项目。
2. 使用 TypeScript。
3. 使用 Tailwind CSS。
4. 创建基础页面：主页、文章页、登录页、仪表盘。
5. 本地运行开发服务器。

当前临时路线：由于本机还没有 `npm`，已先创建静态网站版本，确保可以本地预览和免费上线。等 `npm` 环境准备好后，再升级成 Next.js。

## 1.5 免费上线静态版本

目标：先让别人能通过一个网址访问。

计划：

1. 把当前代码提交到 Git。
2. 创建 GitHub 仓库。
3. 推送代码到 GitHub。
4. 开启 GitHub Pages。
5. 得到免费访问网址。

## 2. 接入 Supabase

目标：网站支持登录和保存数据。

需要创建的数据表：

- `profiles`: 用户资料和角色
- `invitations`: 邀请码
- `posts`: 文章
- `documents`: 可协作文档
- `document_permissions`: 文档授权
- `todos`: 待办事项
- `timer_sessions`: 计时记录
- `diary_entries`: 私密日记
- `files`: 上传文件记录

## 3. 权限模型

角色：

- `owner`: 网站主人，也就是你
- `member`: 受邀用户

内容可见性：

- `public`: 所有人可见
- `members`: 登录用户可见
- `private`: 仅作者本人可见
- `shared`: 指定用户可见或可编辑

核心规则：

- 公开文章任何人可读。
- 私密日记只有作者本人可读写。
- 用户只能编辑自己的文章。
- 你可以授权某个用户编辑你的指定文档。
- 注册必须通过邀请码或管理员审批。

## 4. 完整动态网站上线

目标：升级到 Next.js + Supabase 后，别人可以通过域名访问动态网站。

计划：

1. 把代码推到 GitHub。
2. 在 Vercel 导入项目。
3. 设置环境变量。
4. 部署生产版本。
5. 在域名商购买域名。
6. 把域名 DNS 指向 Vercel。
7. 开启 HTTPS。

## 5. 维护

上线后要做：

- 定期备份 Supabase 数据。
- 检查 Supabase Row Level Security。
- 文件上传限制大小和类型。
- 管理员页面不暴露敏感信息。
- 每次新增功能先本地测试，再上线。
