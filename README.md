# Personal Website

这是一个从零开始构建的个人网站项目。目标不是单纯博客，而是一个带账号、邀请协作、文章/文档空间、PDF 笔记存储、待办计时和私密日记的个人知识空间。

## 推荐技术路线

- 前端和后端框架：Next.js
- 数据库、登录、权限、文件存储：Supabase
- 上线部署：Vercel
- 域名：Porkbun、Namecheap、Cloudflare Registrar 等域名商任选其一

这套组合适合个人项目：开始免费或低成本，后面可以逐步升级，也不用自己维护服务器。

## 第一阶段目标

先做一个能上线的最小版本：

- 首页
- 公开文章列表
- 单篇文章页面
- 管理员登录
- 管理员写文章
- 部署到 Vercel
- 绑定自己的域名

当前已经完成一个静态首页雏形，可以先免费部署到 GitHub Pages。部署步骤见 `DEPLOY.md`。

Next.js 版本本地运行：

```bash
npm install
npm run dev
```

然后打开：

```text
http://localhost:3000
```

## 第二阶段目标

加入账号和邀请制：

- 用户登录
- 邀请码注册
- 管理员审批用户
- 用户个人空间
- 用户只能编辑自己的文章
- 被授权用户可以编辑指定文档

## 第三阶段目标

加入个人工具：

- PDF/笔记上传和分类
- 今日待办事项
- 任务计时器
- 私密日记
- 权限分级：公开、登录可见、仅自己可见、指定用户可编辑

## 第四阶段目标

打磨成长期可用的网站：

- 搜索
- 标签
- 草稿和发布状态
- 自动备份
- 访问统计
- 移动端体验优化
- 安全规则检查

## 开发原则

- 每次只做一小步，保证能运行。
- 先做最小可用版本，再增加复杂功能。
- 所有私密数据都必须有权限控制。
- 上传文件不直接放在代码仓库里，使用 Supabase Storage 或其他对象存储。
- 上线前必须检查环境变量、数据库权限和文件访问权限。
